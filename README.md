# Champion Management API

API REST desenvolvida com Java e Spring Boot para gerenciamento de campeões, criada com foco em prática de back-end e consolidação de conceitos fundamentais de desenvolvimento de software.

## Objetivo

Este projeto foi desenvolvido com o objetivo de praticar a construção de APIs REST utilizando Java e Spring Boot, aplicando conceitos de orientação a objetos, organização em camadas, validação de dados e boas práticas de desenvolvimento.

## Tecnologias utilizadas

- Java 17
- Spring Boot
- Spring Web
- Spring Data JPA
- Bean Validation
- H2 Database
- Maven
- Git e GitHub
- Postman

## Funcionalidades

- Cadastro de campeões
- Listagem de registros
- Busca de informações
- Atualização de dados
- Remoção de registros
- Validação de dados de entrada
- Tratamento de requisições HTTP

## Estrutura do projeto

A aplicação foi organizada seguindo uma arquitetura em camadas, separando as responsabilidades em componentes como:

- Controller
- Service
- Repository

Essa estrutura foi utilizada para manter o código mais organizado, legível e de fácil manutenção.

## Aprendizados aplicados

Durante o desenvolvimento deste projeto, foram praticados os seguintes conceitos:

- Programação Orientada a Objetos
- Criação de APIs REST
- Operações CRUD
- Validação de dados
- Organização de código em camadas
- Versionamento com Git
- Testes de endpoints com Postman

## Como executar o projeto

### Pré-requisitos

- Java 17
- Maven

### Passos

1. Clone este repositório:
   `git clone https://github.com/rafaelalves3070/champion-management-api.git`

2. Acesse a pasta do projeto:
   `cd champion-management-api`

3. Execute a aplicação:
   `./mvnw spring-boot:run`

   Ou, se estiver no Windows:
   `mvnw.cmd spring-boot:run`

4. A API estará disponível localmente em:
   `http://localhost:8080`

## Testes

Os endpoints da aplicação podem ser testados com ferramentas como o Postman, permitindo validar requisições, respostas HTTP e comportamento das operações da API.
